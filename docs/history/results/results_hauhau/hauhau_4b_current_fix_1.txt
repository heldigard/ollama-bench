GOAL: Resolve authentication failures occurring during user login by ensuring valid credentials trigger a successful session creation and invalid credentials result in immediate logout without redirect loops or error masking.

FILES: `src/auth/login.service.ts`, `src/routes/api.auth.routes.ts`, `frontend/src/components/LoginForm.vue` (or equivalent), `.env.production`.

STEPS:
1. Inspect the login flow logic in `login.service.ts`: Verify that password hashing uses bcrypt with a cost factor of 12 and compare against stored hashes using `bcrypt.compareSync()`. Ensure no race conditions exist where two simultaneous requests for the same user ID overwrite each other's session tokens before completion.
2. Audit database queries: Confirm that the login query correctly joins the `users` table with the `sessions` table to check if an active session exists, and ensure foreign key constraints are enforced properly.
3. Update error handling in `api.auth.routes.ts`: Implement specific HTTP status codes (401 Unauthorized for bad credentials, 200 OK on success) instead of generic 500 errors or JSONP responses that may mask the root cause. Ensure JWT tokens are signed with a valid secret key and have an appropriate expiration time (`exp` claim).
4. Refactor frontend logic: Modify `LoginForm.vue` to display specific error messages based on HTTP status codes (e.g., "Invalid username/password" vs "Account locked") and ensure it clears the input fields upon failure while preserving focus for retry efficiency.

EDGE CASES:
- Concurrent login attempts from different devices by the same user must not invalidate each other unless explicitly configured; if mutual exclusivity is required, implement a token revocation mechanism before issuing new tokens.
- Handle cases where the database connection times out during authentication to prevent hanging requests and ensure graceful degradation with retry logic in `login.service.ts`.
- Ensure that API keys or OAuth providers do not bypass local password verification when used as an alternative login method.

ACCEPTANCE:
1. Login with a valid username/password results in immediate redirection to the dashboard, successful session storage (cookies/localStorage), and no console errors related to auth failures.
2. Login attempts using incorrect credentials return HTTP 401 within <500ms and display "Invalid credentials" on the frontend without redirecting back to the login page immediately unless configured for retry limits.
3. Running `npm run test` passes all existing unit tests, specifically those covering authentication edge cases (invalid input, concurrent requests).
