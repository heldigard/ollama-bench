GOAL
Implement a client-side CSV export feature for data displayed in the 'Reports' section of the application. The solution must generate a downloadable `.csv` file containing all rows currently rendered or paginated within that specific report view, formatted with UTF-8 encoding and proper escaping for special characters (commas, quotes, newlines).

FILES
1. `src/pages/ReportsPage.tsx` (or equivalent component): To inject the export trigger button and handle data collection.
2. `src/utils/exportUtils.ts`: To encapsulate CSV generation logic (headers + rows) to avoid duplication across reports.
3. `public/assets/icons/download-csv.svg`: Add a visual icon for better UX if not already present in an SVG sprite or library.

STEPS
1. **Create Utility Function**: In `src/utils/exportUtils.ts`, define a function `generateCSV(data: any[], headers: string[])` that returns a Blob object with MIME type `text/csv;charset=utf-8`. Ensure the function handles edge cases like empty arrays, missing columns, and special characters (wrap fields in double quotes if they contain commas or newlines).
2. **Add Export Button**: In `ReportsPage`, locate the main action toolbar or header area. Add a button with an icon labeled "Export CSV". Bind this to a handler function that calls `generateCSV`.
3. **Data Preparation Logic**: Inside the export handler:
   - Determine if data is paginated. If so, fetch all available pages (or implement server-side aggregation) before generating the file, OR ensure the current view's dataset includes pagination metadata in headers/rows as agreed upon by product requirements. Assume full dataset retrieval for this spec unless specified otherwise.
   - Extract column names from the table definition or API response keys to serve as CSV headers.
4. **Trigger Download**: Create a temporary `<a>` element with `href` set to `URL.createObjectURL(blob)` and click it programmatically, then revoke the object URL after 100ms-500ms to free memory.
5. **Loading State & Feedback**: Add a loading spinner or disable the button during data fetching/blob creation. Show a toast notification ("Report downloaded successfully") upon completion.

EDGE CASES
- **Empty Dataset**: If no rows exist, export only headers (or an empty file with just newlines) without crashing.
- **Special Characters**: Ensure fields containing commas, double quotes, or line breaks are properly escaped according to CSV standards (e.g., `value"with` becomes `"value""with"`).
- **Large Datasets**: If the report has thousands of rows, implement chunked reading for browser memory safety if using large text blobs, though standard Blob creation usually handles up to ~10MB well. Consider server-side streaming for >50k rows.
- **Browser Compatibility**: Ensure `URL.createObjectURL` and `Blob` work across target browsers (fallback to `createObjectURL` or string URI methods if needed).
- **Column Order/Nulls**: Maintain the original column order from the UI/API; handle null values by converting them to empty strings in the CSV output.

ACCEPTANCE
1. Clicking "Export CSV" downloads a file named `<ReportName>_YYYYMMDD_HHMMSS.csv`.
2. Opening the downloaded file in Excel/Google Sheets displays data exactly as seen on screen, with correct headers and no broken text wrapping due to special characters.
3. If 0 rows are displayed, the download contains only the header row (or an empty line if strictly following "no data" convention).
4. The button shows a loading state while processing large datasets (>1k rows) without freezing the UI for more than 2 seconds.
5. A success toast message appears after the file is generated and downloaded.
