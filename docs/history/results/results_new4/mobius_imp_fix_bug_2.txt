GOAL: Identify and resolve a security vulnerability or functional failure within the authentication system (e.g., session hijacking, incorrect password hashing, JWT expiration issues, or unauthorized access to protected routes).

FILES: 
- `auth_middleware.js` / `auth_controller.js` (Authentication logic)
- `.env` (Environment variables for secrets/keys)
- `package.json` (Dependency check for outdated security libraries)
- `user_model.js` (User schema and password hashing configuration)

STEPS:
1. Audit the authentication flow: Trace a request from login/registration to session establishment.
2. Identify specific bug type: Determine if it is a logic error (e.g., "always returns true"), a security flaw (e.g., weak salt), or a state issue (e.g., stale tokens).
3. Reproduce the bug: Create a test script or manual steps to consistently trigger the failure.
4. Implement fix: Apply code changes (e.g., update `bcrypt` rounds, validate JWT signatures, or enforce MFA).
5. Verify fix: Run unit tests for success and failure paths; perform a penetration test on the affected endpoint.

EDGE CASES:
- Expired tokens/sessions being accepted.
- Brute force attempts not triggering rate limiting.
- SQL injection vulnerabilities in login fields.
- Handling of null or undefined user objects during session retrieval.
- Concurrent logins from multiple devices.

ACCEPTANCE:
- Authentication succeeds only with valid credentials.
- Unauthorized users receive a 401/403 status code for protected resources.
- Passwords are never stored in plain text (verified via DB inspection).
- JWTs/Sessions have a defined expiration and are properly revoked on logout.
- Unit tests pass with 100% coverage on the auth module.
