GOAL
Implement a CSV export feature for all data rows displayed in the `/reports` view. The exported file must contain only columns currently visible in the UI, sorted by default sort order (descending), and formatted with standard comma delimiters.

FILES
- `src/components/ReportsTable.tsx`: Add "Export to CSV" button component and export logic hook.
- `src/hooks/useReportData.ts`: Update data fetching or filtering hooks if necessary to support column selection state for the export function.
- `public/reports.csv` (optional): Create a sample file in tests directory only; do not commit production code here unless running CI.

STEPS
1.  **UI Component**: Add an "Export CSV" button next to the primary action buttons on the Reports page header (`<ReportsPage>`). The button should be disabled if no data is loaded or if all columns are hidden (optional, but recommended for UX).
2.  **Data Preparation Hook**: Create a function `exportToCSV(data: ReportRow[], visibleColumns: string[])` within `useReportData.ts`. This function must:
    *   Map over the provided rows and selected columns to generate an array of strings.
    *   Handle date formatting (convert JS Date objects or ISO strings to YYYY-MM-DD).
    *   Escape any commas, quotes, or newlines within cell values using RFC 4180 standards.
3.  **File Generation**: Use the `Blob` API and a temporary anchor tag (`<a>` element) with `download="true"` attribute to trigger the browser's native download dialog without requiring external libraries like `xlsx`.
4.  **Integration**: Wire the button click event to call the export function, passing the current state of report data and visible columns from the UI context.

EDGE CASES
-   **Empty Dataset**: If no reports are loaded, disable the button or show a toast notification "No data available for export."
-   **Special Characters**: Ensure values containing commas (e.g., `"New York, NY"`) do not break the CSV parsing logic by wrapping them in double quotes.
-   **Date Formats**: Verify that timezone-aware dates are converted to local time or UTC consistently based on application settings before exporting.
-   **Column Filtering**: If a user filters the report (e.g., "Active Users Only"), ensure only rows matching the filter criteria and columns currently visible in the filtered view are exported, not all historical data unless explicitly requested via an advanced export modal.

ACCEPTANCE
1.  Clicking the button downloads `filename.csv` to the device's default download folder.
2.  Opening the downloaded file reveals headers corresponding exactly to the column names displayed on screen at that moment.
3.  The exported rows match the visible data in the table, excluding hidden columns and filtered-out records (if applicable).
4.  No console errors are thrown during export operations involving special characters or large datasets (>10k rows).
