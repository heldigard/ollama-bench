GOAL
Optimize frontend performance to reduce Time to First Byte (TTFB) and Largest Contentful Paint (LCP) by 40% for the main user dashboard without degrading functionality or increasing bundle size.

FILES
- `src/components/Dashboard/index.tsx` (Main dashboard component tree)
- `public/images/*.png`, `/assets/icons/svg` (Static assets currently loaded via public path)
- `package.json` (To verify dependency versions and build scripts)
- `webpack.config.js` or `vite.config.ts` (Bundle configuration for optimization settings)

STEPS
1. **Audit Current Bundle Size**: Run a bundle analysis tool (e.g., webpack-bundle-analyzer via CLI) to identify the top 3 largest modules in the dashboard tree and their dependency chains.
2. **Implement Code Splitting with Dynamic Imports**: Replace static imports of heavy charting libraries or data-fetcher functions within `Dashboard` components with dynamic `import()` syntax, triggering lazy loading only when specific sub-views are accessed.
3. **Optimize Asset Delivery**: 
   - Convert all PNG assets to WebP format where supported by the browser (using a tool like svgo/convert).
   - Implement responsive image sources (`srcset`) for dashboard thumbnails and icons based on viewport width.
4. **Enable Resource Hints**: Configure build tools to automatically inject `<link rel="preload">` for critical CSS required for initial layout rendering and `<link rel="prefetch">` for secondary API endpoints used in the dashboard sidebar.
5. **Minimize Network Requests**: Consolidate multiple small font files into a single `woff2` file using Google Fonts or similar CDN strategies, reducing HTTP requests by at least 30%.

EDGE CASES
- **Slow Client Devices**: Ensure lazy-loaded components have fallback UI states (e.g., skeleton loaders) to prevent layout shift and maintain perceived performance on low-end devices.
- **Offline Mode**: Verify that cached assets are served correctly via Service Worker logic if the dashboard is accessed without an active internet connection after initial load optimization.
- **CORS Issues**: Ensure dynamic imports do not inadvertently trigger CORS errors when fetching data from third-party analytics or mapping APIs during lazy loading.

ACCEPTANCE
- LCP (Largest Contentful Paint) score improves to 2.5s or lower on a standard mobile network connection (3G).
- Initial JavaScript bundle size is reduced by at least 40% compared to the baseline build.
- No runtime errors occur during lazy loading of sub-components in the dashboard view.
