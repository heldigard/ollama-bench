GOAL
Implement a client-side CSV export feature for the existing Reports Page that generates and downloads a file containing all data currently displayed in the report's main table without triggering full-page reloads. The solution must handle dynamic rendering, large datasets (with pagination or chunking if necessary), proper character encoding (UTF-8 with BOM to ensure Excel compatibility), and escape special characters within cells to prevent CSV parsing errors.

FILES
1. `src/pages/reports/ReportsPage.tsx` (or equivalent main component file): Add export button, trigger download logic, handle loading states during generation.
2. `src/utils/export/csvExporter.ts`: Create a utility function that accepts an array of objects and returns a Blob containing the CSV content with headers and proper escaping.
3. `public/locales/en.json` (optional but recommended): Add translation keys for "Export to CSV", "Generating...", etc., if using i18n.

STEPS
1. Define the data structure: Identify the exact shape of the report table rows (e.g., an array of objects with stringifiable properties). Ensure all columns are included in the export, even those hidden from view unless specified otherwise.
2. Implement `csvExporter.ts`:
   - Create a function `generateCSV(data: any[]): Blob`.
   - Extract headers as keys of the first object (or pass explicit header config if schema varies per report type).
   - Map each row to an array of strings, applying CSV escaping rules: wrap fields containing commas/quotes/newlines in double quotes and escape internal quotes by doubling them.
   - Use `Blob` with MIME type `"text/csv;charset=utf-8"` (include BOM `\uFEFF` at the start if Excel compatibility is required).
3. Update ReportsPage Component:
   - Add an "Export" button next to existing action buttons or in a dropdown menu, using consistent iconography and styling from the design system.
   - On click, call `generateCSV(currentReportData)` where `currentReportData` represents the currently filtered/paginated dataset (or fetch all if pagination is client-side).
   - Create an anchor element (`<a>`) with a dynamically generated filename (e.g., `report_YYYYMMDD_HHMMSS.csv`).
   - Programmatically click the anchor to trigger download, then remove it from DOM.
4. Handle Loading States: Show a spinner or toast notification while generating large files (>50k rows) if processing happens asynchronously; for small datasets (<10k), synchronous generation is acceptable.
5. Add Error Handling: Wrap export logic in try-catch to handle cases where data might be malformed (e.g., circular references, non-stringifiable types). Display user-friendly error messages via toast/alerts.

EDGE CASES
- **Large Datasets**: If the report displays >10k rows, consider chunking or virtualization; otherwise, warn users about potential browser memory limits before export.
- **Special Characters**: Ensure newlines (`\n`), commas (`,`), and double quotes (`"`) inside cell values are correctly escaped per RFC 4180 CSV standards to avoid breaking row delimiters in Excel/Sheets.
- **Empty Data Set**: Handle cases where the report table is empty by exporting a header-only file or showing an "No data found" message before triggering download.
- **Browser Compatibility**: Test on Chrome, Firefox, Safari, and Edge; note that some browsers may require user interaction (click) to trigger blob downloads due to autoplay policies.
- **Date/Time Formatting**: Ensure date columns are formatted as strings in a locale-independent format (e.g., `YYYY-MM-DD`) or ISO 8601 to prevent Excel misinterpreting them as dates with timezones.

ACCEPTANCE
- User clicks "Export" button and downloads
