GOAL: Implement a CSV export functionality for the data table displayed on the Reports page, allowing users to download the current filtered view of report data as a .csv file.

FILES:
- `src/pages/ReportsPage.tsx` (or equivalent): Main UI component for reports.
- `src/components/ReportTable.tsx`: Component rendering the data grid.
- `src/utils/exportHelpers.ts`: New utility file for CSV generation logic.
- `src/hooks/useExportData.ts`: Optional hook to manage export state and formatting.

STEPS:
1. Create a utility function in `src/utils/exportHelpers.ts` that takes an array of objects and returns a Blob containing the CSV string. 
   - Include logic to handle commas within cell values (wrap in quotes).
   - Map keys from the first object to the header row.
2. Add an "Export to CSV" button to the `ReportTable` component's action bar/header.
3. Implement a click handler for the button that:
   - Retrieves the current filtered and sorted data from the state.
   - Calls the utility function to generate the Blob.
   - Triggers a browser download using `URL.createObjectURL`.
4. Ensure the export respects active filters (e.g., date range, category) and pagination (export only visible page vs. all filtered results - *clarify with stakeholder, default to all filtered*).
5. Style the button to match existing UI components (e.g., using a standard primary or secondary button style).

EDGE CASES:
- Empty Data State: If no records match filters, show a "No data available to export" toast/notification instead of an empty file.
- Special Characters: Ensure non-ASCII characters and emojis are handled correctly in the CSV encoding (UTF-8 with BOM).
- Large Datasets: For datasets > 5000 rows, consider implementing a loading state or processing via a Web Worker to prevent UI freezing.
- Column Selection: If users can toggle visible columns, ensure only visible columns are included in the export.

ACCEPTANCE:
- User clicks "Export to CSV" and a file named `report_export_[timestamp].csv` is downloaded.
- The exported CSV contains all rows currently filtered by the user's active UI filters.
- Data types (Dates, Currencies) are formatted correctly in the CSV (e.g., YYYY-MM-DD).
- The button is disabled or hidden if no data is present in the table.
