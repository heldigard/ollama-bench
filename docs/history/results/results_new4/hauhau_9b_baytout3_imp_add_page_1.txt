GOAL
Implement a client-side CSV export feature for the existing Reports Page that allows users to download their current view (filtered and sorted data) as a `.csv` file. The implementation must handle dynamic table rendering, preserve column headers from the UI, manage large datasets via chunked processing or virtualization if applicable, and ensure proper encoding/character set handling (UTF-8 with BOM for Excel compatibility).

FILES
1. `src/pages/reports/page.tsx` (or equivalent main component file) - Add export button logic and trigger handler.
2. `src/components/ui/table/TableExportButton.tsx` (New or existing wrapper) - Create a reusable, accessible download icon/button with tooltip "Download CSV".
3. `src/lib/csv-export/utils.ts` (New) - Implement helper functions for header extraction, data transformation to string rows, and file blob creation using the native File API.
4. `public/assets/icons/download-csv.svg` (Optional/Existing) - Ensure an SVG icon is available or create one with semantic meaning (arrow-down + csv-file).

STEPS
1. **Header Extraction**: Create a utility function that accepts the current table's column definitions (from props/state) and returns an array of header strings matching exactly what renders in the `<thead>`.
2. **Data Transformation**: Implement a transformation step that maps the current row data objects into flat arrays, ensuring all columns present in the UI are included even if some values are `null` or empty strings (convert to string representation). Handle date/time fields by formatting them according to existing locale settings before serialization.
3. **Blob Creation**: Use `new Blob([csvContent], { type: 'text/csv;charset=utf-8;' })`. Ensure the content starts with a UTF-8 BOM (`\uFEFF`) if Excel compatibility is required, or standard CSV format otherwise (prefer BOM for cross-platform consistency).
4. **Trigger Download**: Create an anchor element `<a>` programmatically attached to `document.body`, set its `href` to `URL.createObjectURL(blob)`, assign the filename as `${reportName}_${timestamp}.csv`, and trigger a click event before removing the node from DOM after download starts.
5. **UI Integration**: Add a "Download CSV" button next to existing table actions (e.g., Print, Share). Ensure it respects current filter/sort state by capturing data *after* these operations are applied but *before* pagination if virtualization is used (or handle full dataset export with progress indication for >10k rows).
6. **Accessibility**: Add `aria-label="Download report as CSV"` and keyboard focus management to the new button.

EDGE CASES
- **Empty Dataset**: If no data matches filters, download a file containing only headers or an empty row (decide on behavior; prefer header-only with note in filename).
- **Special Characters**: Escape double quotes within cell values by doubling them (`"` -> `""`) and wrap fields containing commas/quotes/newlines in single quotes.
- **Large Datasets (>10k rows)**: Implement streaming or chunked writing to memory if browser limits are hit, or show a progress toast during export for datasets >5MB.
- **Date/Time Formats**: Ensure dates match the UI locale (e.g., `MM/DD/YYYY` vs `DD/MM/YYYY`) before serialization; avoid raw ISO strings unless configured otherwise.
- **Browser Compatibility**: Test on Safari (requires BOM), Chrome, Firefox, and Edge to ensure Excel opens without extra columns or encoding errors.

ACCEPTANCE
1. User clicks "Download CSV" button while viewing a filtered/sorted report table.
2. Browser downloads a file named `ReportName_YYYYMMDD_HHMMSS.csv` (or configurable naming).
