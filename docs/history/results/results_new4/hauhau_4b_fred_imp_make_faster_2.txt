GOAL
Optimize frontend performance to reduce Time to Interactive (TTI) and Largest Contentful Paint (LCP) by 40% for the main application dashboard without degrading user experience or breaking existing functionality.

FILES
- `src/components/Dashboard/index.tsx` - Main dashboard container component.
- `src/hooks/useDashboardData.ts` - Data fetching logic using React Query/SWR.
- `src/assets/images/*.png`, `*.svg` - All static assets used in the UI.
- `package.json` - To verify dependency versions for potential tree-shaking opportunities.

STEPS
1. **Audit Current Performance**: Run Lighthouse audit and identify specific bottlenecks (e.g., render-blocking resources, excessive initial bundle size).
2. **Implement Code Splitting with Dynamic Imports**: Replace static imports in `Dashboard/index.tsx` for heavy chart libraries or data tables with dynamic `import()` calls to load them only when visible on screen.
3. **Optimize Data Fetching Strategy**: Modify `useDashboardData.ts` to:
    - Implement aggressive caching strategies (e.g., stale-while-revalidate).
    - Add debouncing for search/filter inputs if applicable.
4. **Asset Optimization**: Compress all images in the `/assets/images` directory using WebP format where supported and implement lazy loading (`loading="lazy"`) or Intersection Observer triggers for off-screen elements.
5. **Bundle Analysis & Tree Shaking**: Run `bundle-analyzer` to identify unused dependencies; remove them from `package.json` if confirmed unused, ensuring the final bundle size is under 100KB (gzipped).

EDGE CASES
- Network throttling scenarios: Ensure fallback data or skeleton screens load within 2 seconds even on slow connections.
- Large dataset rendering: If dashboard filters reveal >50 records, implement virtual scrolling to prevent memory spikes during render.
- Browser compatibility: Verify that dynamic imports and WebP formats do not break support for older browsers (e.g., IE11) if still required; otherwise, add a graceful degradation message.

ACCEPTANCE
- Lighthouse Performance score improves from current baseline by at least 20 points.
- Time to Interactive reduces by >4 seconds on the primary production environment.
- No regressions in unit tests for `Dashboard` and data fetching hooks.
- All images render correctly with optimized formats (WebP) or fallbacks where necessary.
