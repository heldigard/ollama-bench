GOAL: Optimize the frontend and backend performance of the main dashboard to reduce Time to Interactive (TTI) and Largest Contentful Paint (LCP). The target is a 50% reduction in initial load time.

FILES:
- /frontend/src/pages/Dashboard.tsx (Main entry point)
- /frontend/src/components/Charts/*.tsx (Data visualization components)
- /frontend/src/hooks/useFetchDashboardData.ts (API interaction logic)
- /backend/api/routes/dashboard.js (Backend endpoint)
- /backend/models/Analytics.js (Database schema)

STEPS:
1. Audit Performance: Run a Lighthouse audit and Chrome DevTools Network tab analysis to identify bottlenecks (e.g., large payloads, blocking JS, or slow DB queries).
2. Implement Pagination/Infinite Scroll: Modify the dashboard data fetcher to request only the first 50 records instead of the entire dataset.
3. Data Aggregation: Update the backend endpoint to perform SQL aggregations (SUM, AVG) rather than fetching raw rows and calculating totals on the client side.
4. Code Splitting & Lazy Loading: Wrap heavy chart components in `React.lazy()` and use dynamic imports to ensure they only load when rendered.
5. Caching Strategy: Implement a Redis cache layer for static dashboard metrics that do not require real-time updates.
6. Image/Asset Optimization: Convert all dashboard icons to SVG and implement lazy loading for non-critical images.

EDGE CASES:
- Large Data Sets: Ensure the UI remains responsive when a user requests a date range spanning >1 year.
- Network Latency: Implement "Skeleton Screens" or loading spinners to provide visual feedback during slow API responses.
- Concurrent Requests: Handle race conditions if a user changes filters rapidly before the previous request completes.

ACCEPTANCE:
- LCP (Largest Contentful Paint) must be under 2.5 seconds on a standard 4G connection.
- Total payload size for the initial dashboard JSON response must be < 100KB.
- Database query execution time for the primary dashboard endpoint must be < 300ms.
- No "Uncaught TypeErrors" related to null data during asynchronous loading states.
