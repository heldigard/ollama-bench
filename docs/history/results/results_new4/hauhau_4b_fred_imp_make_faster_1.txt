GOAL
Optimize frontend performance to reduce Time to First Byte (TTFB) and Largest Contentful Paint (LCP) by implementing code splitting, lazy loading for non-critical components, optimizing asset delivery via CDN with appropriate caching headers, and minifying critical CSS/JS bundles. Target LCP reduction of at least 30% without degrading user experience on secondary pages.

FILES
- `src/index.html`: Entry point script injection logic.
- `src/app.js` (or equivalent main bundle): Application initialization code to be split or deferred.
- `/public/assets/`: Directory containing CSS, JS, and image assets requiring optimization.
- `.eslintrc.json` / `.prettierrc`: Configuration for minification rules if build tools are used.

STEPS
1. **Audit Current Performance**: Run Lighthouse audit on the production environment to identify specific bottlenecks (e.g., render-blocking resources, excessive JavaScript execution). Document top 3 issues.
2. **Implement Code Splitting**: Configure bundler (Webpack/Vite/Rollup) to split the application into chunks based on routes or feature flags. Ensure `index.html` only loads the critical initial chunk (~10-15kb JS).
3. **Lazy Load Non-Critical Assets**: Wrap all non-critical JavaScript modules and heavy image components in dynamic imports (`import()`) within their respective route handlers to defer execution until navigation occurs.
4. **Optimize Asset Delivery**: 
   - Compress CSS/JS files (gzip/brotli).
   - Configure `Content-Security-Policy` headers strictly but permissively for CDN origins.
   - Set long-cache (`max-age=1y, s-maxage`) for static assets and short cache with validation (`no-store`, `immutable`) for versioned bundles to prevent stale resource loading on subsequent visits.
5. **Defer Non-Critical Rendering**: Use the browser's native lazy-loading attribute or a library (e.g., Intersection Observer) to defer image rendering until they enter the viewport.
6. **Verify and Monitor**: Re-run Lighthouse, verify Core Web Vitals in Chrome DevTools Performance tab, and set up Sentry/Google Analytics performance monitoring dashboards for regression tracking.

EDGE CASES
- **Slow Network Environments**: Ensure lazy-loaded chunks do not cause "flash of unstyled content" (FOUC) or infinite loading spinners; implement skeleton screens with fallbacks that load immediately upon initial chunk render.
- **Caching Conflicts**: Verify that aggressive caching headers do not bypass security policies for authenticated dashboard data updates; ensure cache-busting version strings are applied to JS/CSS bundles but excluded from API response caches where necessary.
- **Build Pipeline Failures**: Ensure the build process fails if minification or compression steps produce larger files than source, preventing performance regressions due to inefficient bundling configurations.

ACCEPTANCE
1. Lighthouse "Performance" score improves by at least 20 points (e.g., from 45 to 65).
2. Largest Contentful Paint (LCP) metric drops below 2.5 seconds on mobile and desktop devices respectively.
3. Initial JavaScript bundle size in the browser network tab is reduced by >50% compared to previous build artifacts.
4. No new critical rendering path bottlenecks introduced; all non-critical scripts execute after a visible interactive element appears or via lazy loading triggers.
