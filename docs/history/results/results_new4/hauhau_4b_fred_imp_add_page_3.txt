GOAL
Implement a CSV export feature for all report rows displayed in `/reports` (or `#reports`). The functionality must allow users to download a file containing only data from currently visible or paginated results, formatted as standard RFC 4180 compliant CSV.

FILES
- Frontend: `src/components/ReportsTable.tsx`, `src/hooks/useExportCSV.ts`
- Backend (if required for large datasets): `src/api/reports/exportController.js`
- Assets: `public/templates/report_export_template.csv` (optional, if headers are dynamic)

STEPS
1. **UI Trigger**: Add a "Download CSV" button next to the existing "Export PDF/Excel" controls in the Reports header toolbar (`<Button variant="outline">`).
2. **Data Fetching Hook**: Create or update `useReportFilters` hook to include an optional parameter `{ exportOnly: boolean }`. If true, fetch all rows matching current filters regardless of pagination limits (or implement server-side aggregation).
3. **CSV Generation Logic**: 
   - Construct a header row based on the table column definitions (`columns.map(c => c.title)`).
   - Iterate through fetched data and map each object to an array of values using `columnAccessor`.
   - Handle special characters in cell content (commas, quotes) by wrapping fields containing them in double quotes and escaping internal quotes per RFC 4180.
4. **File Download**: 
   - Create a Blob from the generated CSV string with MIME type `text/csv;charset=utf-8;`.
   - Generate an object URL (`URL.createObjectURL`).
   - Trigger download via `<a>` tag click event or browser API, naming the file `{report_name}_export_{timestamp}.csv`.
5. **Cleanup**: Ensure the Blob is released (using `URL.revokeObjectURL`) after navigation to prevent memory leaks in long-running sessions.

EDGE CASES
- **Empty Dataset**: If no rows match filters, display a "No data available" message and skip file creation logic gracefully without crashing.
- **Special Characters**: Input fields containing newlines (`\n`), commas (`,`), or quotes (`"`) must be properly escaped to prevent CSV parsing errors in Excel/Sheets.
- **Large Datasets (>10k rows)**: Implement server-side streaming or chunked generation if client-side Blob creation risks memory exhaustion; otherwise, enforce a row limit on the export action with an alert message.
- **Pagination**: Ensure exported data reflects *only* currently visible items (if pagination is active) unless "Export All" mode is explicitly selected via UI toggle.

ACCEPTANCE
1. User clicks "Download CSV".
2. Browser downloads `report_name_export_YYYY-MM-DD_HH.MM.ss.csv`.
3. Opening the file in a text editor or spreadsheet shows correct headers and data corresponding to visible table rows.
4. Cells containing commas are enclosed in quotes (e.g., `"Price, Rate"`).
5. No console errors occur during export execution.
