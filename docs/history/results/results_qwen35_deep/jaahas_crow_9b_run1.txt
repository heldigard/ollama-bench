[CRIT] src/orders/service.py: SQL injection vulnerability via direct string interpolation of user_id into query. The `f"SELECT * FROM orders WHERE user_id = '{user_id}'"` pattern allows attackers to inject malicious SQL by providing a crafted user_id value (e.g., `1 OR 1=1`). This should use parameterized queries with `conn.execute(query, {"user_id": user_id})` instead.

[CRIT] src/orders/service.py: Stripe API key exposure risk. `stripe.refund(charge)` is called without any authentication context visible in the diff. If `stripe` is a client initialized elsewhere without proper secret management (env vars, secrets manager), the API key could be hardcoded or leaked. Verify `stripe` is properly configured with secure credentials.
